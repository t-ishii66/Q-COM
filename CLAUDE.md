# Q-CAL: 量子コンピュータ学習プロジェクト

## プロジェクト概要

量子コンピュータの基礎を学び、その理解を物語として記述するプロジェクト。

## 2段階のアプローチ

### 第1段階: ノート作成
物語とは独立した、いつでも参照できる正確な物理ノートを作成する。
物語のストーリーや展開を考える際の参照資料となる。

**ノートの構成（予定）:**
1. 量子ビット — 基本素子、状態の表現
2. 量子ゲート — 基本的な操作（パウリゲート、アダマールゲート、CNOTなど）
3. 量子回路 — ゲートの組み合わせ、回路の読み方
4. 量子もつれ（エンタングルメント）
5. 量子テレポーテーション — 応用的な話題へ発展

**ノートの配置:** `notes/` ディレクトリ

### 第2段階: 物語化
第1段階で作成したノートをベースに、量子コンピュータの概念を物語として記述する。

## プロジェクト方針

- **言語**: 日本語
- **対象読者**: 大学1〜2年生（物理学・情報系の学生）
- **ノート形式**: Markdown
- **物語のスタイル**: 物理的な記述を主軸に、登場人物の会話を織り交ぜる形式
  - 物理内容はまとめてわかりやすく記述する（教科書的な正確さを保つ）
  - 登場人物が実験を進めていく流れの中で、時折セリフが入る
  - 会話で進めすぎると物理から離れ、物理だけだと参考書と変わらなくなるため、そのバランスを取る

## ビット順の規約

`notes/`（教科書の世界）と `notebooks/`（Qiskit の世界）ではビット順・インデックスが異なる。

| | `notes/`（教科書） | `notebooks/`（Qiskit） |
|---|---|---|
| インデックス | 1 始まり | 0 始まり |
| ケット表記 | $\vert q_1 q_2 q_3\rangle$（$q_1$ が最上位） | $\vert q_2 q_1 q_0\rangle$（$q_0$ が最下位） |
| ビット順 | ビッグエンディアン | リトルエンディアン |

**原則:** `notebooks/` では Qiskit のビット順をそのまま使う。教科書のビット順に合わせる処理は入れない。

**例外:** Qiskit のビット順のまま計算を続けると、`notes/` 側の理論と辻褄が合わなくなる箇所がある（例: QPE の位相計算、DFT 行列 $F_N$ との数値比較）。その箇所に限り、教科書のビット順を意識したコードを書く。その際は必ずコード中のコメントおよびマークダウンの解説で「教科書のビット順に合わせている」ことを明示する。

**逆に言えば:** 何も注釈がなければ、そのコードは Qiskit のビット順で処理している。

## notes/ と notebooks/ の役割分担

- **`notes/`**: 理論の解説（物理・数学）。教科書的な記述。
- **`notebooks/`**: Qiskit による実装と実験。理論だけでなく、Qiskit の API・Python の構文・データ構造など**実作業に必要な知識**も解説する。初めて登場する関数やクラスには、それが何をするものかの説明を付ける。

- **Bob**: 男の子、12歳
- **Charlie**: 男の子、11歳
